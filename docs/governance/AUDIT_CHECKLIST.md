## 3. الضوابط التقنية

### 3.1 الهوية والوصول (Identity & Access)

```
□ المصادقة متعددة العوامل (MFA)
  ├─ □ مُفعل لجميع المستخدمين
  ├─ □ مُفعل للوصول للإنتاج
  ├─ □ مُفعل لـ GitHub/SCM
  └─ □ مُفعل لقاعدة البيانات

□ إدارة الهوية
  ├─ □ نظام SSO مركزي
  ├─ □ مراجعة الوصول دورياً
  ├─ □ إلغاء الوصول الفوري عند المغادرة
  └─ □ سياسة كلمات المرور مُطبقة

□ الحد الأدنى من الامتيازات (PoLP)
  ├─ □ RBAC مُطبق
  ├─ □ الأدوار موثقة
  ├─ □ فصل الواجبات
  └─ □ مراجعة الصلاحيات ربع سنوية
```

**الأدلة المطلوبة:**
- [ ] لقطات شاشة لإعدادات MFA
- [ ] قائمة الأدوار والصلاحيات
- [ ] سجلات مراجعة الوصول
- [ ] إجراءات إلغاء الوصول

### 3.2 التشفير (Encryption)

```
□ التشفير أثناء النقل
  ├─ □ TLS 1.2+ لجميع الاتصالات
  ├─ □ شهادات SSL صالحة
  ├─ □ HSTS مُفعل
  └─ □ Certificate pinning (للتطبيقات)

□ التشفير أثناء التخزين
  ├─ □ قاعدة البيانات مشفرة
  ├─ □ النسخ الاحتياطية مشفرة
  ├─ □ الملفات الحساسة مشفرة
  └─ □ مفاتيح مُدارة بشكل آمن

□ إدارة المفاتيح
  ├─ □ KMS أو HSM مستخدم
  ├─ □ تدوير المفاتيح دوري
  ├─ □ فصل بيئات المفاتيح
  └─ □ نسخ احتياطي للمفاتيح
```

**الأدلة المطلوبة:**
- [ ] تقرير SSL Labs (A+ rating)
- [ ] إعدادات تشفير قاعدة البيانات
- [ ] سياسة تدوير المفاتيح
- [ ] تكوين KMS/Vault

### 3.3 المراقبة والتسجيل (Monitoring & Logging)

```
□ التسجيل الأمني
  ├─ □ جميع محاولات الدخول مُسجلة
  ├─ □ العمليات الحساسة مُسجلة
  ├─ □ تغييرات الصلاحيات مُسجلة
  └─ □ الأخطاء الأمنية مُسجلة

□ جمع وتحليل السجلات
  ├─ □ SIEM مُنشر
  ├─ □ تنبيهات مُكونة
  ├─ □ لوحات مراقبة
  └─ □ الاحتفاظ بالسجلات (سنة+)

□ المراقبة النشطة
  ├─ □ مراقبة 24/7
  ├─ □ تنبيهات الحوادث
  ├─ □ مراقبة الأداء
  └─ □ مراقبة الاستخدام الشاذ
```

**الأدلة المطلوبة:**
- [ ] عينة من سجلات التدقيق
- [ ] تكوين التنبيهات
- [ ] لقطات شاشة SIEM
- [ ] تقرير الاحتفاظ بالسجلات

### 3.4 أمان التطبيق (Application Security)

```
□ دورة التطوير الآمن (SDLC)
  ├─ □ مراجعة الكود إجبارية
  ├─ □ SAST في CI/CD
  ├─ □ SCA للتبعيات
  ├─ □ Secret scanning
  └─ □ DAST للتطبيق

□ البوابات الأمنية
  ├─ □ لا يمكن تخطي الفحوصات
  ├─ □ فشل تلقائي عند الثغرات الحرجة
  ├─ □ موافقة أمنية للنشر
  └─ □ فحص Docker images

□ حماية التطبيق
  ├─ □ WAF مُفعل
  ├─ □ Rate limiting
  ├─ □ Input validation
  ├─ □ Output encoding
  └─ □ CSRF protection
```

**الأدلة المطلوبة:**
- [ ] CI/CD pipeline configuration
- [ ] تقارير SAST/SCA
- [ ] نتائج اختبار الاختراق
- [ ] تكوين WAF

### 3.5 أمان البنية التحتية (Infrastructure Security)

```
□ حماية الشبكة
  ├─ □ Firewall rules موثقة
  ├─ □ Network segmentation
  ├─ □ VPN للوصول الإداري
  └─ □ IDS/IPS مُفعل

□ حماية الخوادم
  ├─ □ Hardening مُطبق
  ├─ □ التحديثات الأمنية منتظمة
  ├─ □ Anti-malware
  └─ □ مراقبة سلامة الملفات

□ حماية الحاويات (إن وجدت)
  ├─ □ Base images آمنة
  ├─ □ فحص الثغرات
  ├─ □ Runtime protection
  └─ □ Registry security
```

**الأدلة المطلوبة:**
- [ ] مخطط الشبكة
- [ ] قواعد Firewall
- [ ] تقرير Vulnerability scan
- [ ] سياسة التحديثات

## 4. ضوابط العمليات

### 4.1 إدارة التغيير (Change Management)

```
□ عملية التغيير
  ├─ □ جميع التغييرات موثقة
  ├─ □ الموافقات مطلوبة
  ├─ □ اختبار قبل النشر
  ├─ □ Rollback plan
  └─ □ مراجعة ما بعد التغيير

□ إدارة الإصدارات
  ├─ □ Version control (Git)
  ├─ □ Branching strategy
  ├─ □ Release process
  └─ □ Changelog maintained
```

**الأدلة المطلوبة:**
- [ ] عملية التغيير الموثقة
- [ ] عينة من طلبات التغيير
- [ ] سجل الإصدارات

### 4.2 إدارة الثغرات (Vulnerability Management)

```
□ اكتشاف الثغرات
  ├─ □ فحص دوري (أسبوعي)
  ├─ □ اختبار اختراق (سنوي)
  ├─ □ Bug bounty program
  └─ □ مراقبة CVEs

□ معالجة الثغرات
  ├─ □ SLA للإصلاح (حسب الخطورة)
  ├─ □ تتبع حتى الإغلاق
  ├─ □ توثيق الإصلاحات
  └─ □ التحقق بعد الإصلاح

□ جدول الإصلاح
  ├─ □ Critical: 24 ساعة
  ├─ □ High: 7 أيام
  ├─ □ Medium: 30 يوم
  └─ □ Low: 90 يوم
```

**الأدلة المطلوبة:**
- [ ] تقارير الفحص الدوري
- [ ] تقرير اختبار الاختراق
- [ ] سجل الثغرات والإصلاحات
- [ ] إحصائيات وقت الإصلاح

### 4.3 إدارة الحوادث (Incident Management)

```
□ خطة الاستجابة
  ├─ □ IRP موثقة ومحدثة
  ├─ □ الأدوار محددة
  ├─ □ جهات الاتصال محدثة
  └─ □ Runbooks جاهزة

□ تمارين الاستجابة
  ├─ □ Tabletop exercises (ربع سنوي)
  ├─ □ Technical drills (نصف سنوي)
  ├─ □ Full simulation (سنوي)
  └─ □ توثيق الدروس المستفادة

□ سجل الحوادث
  ├─ □ جميع الحوادث موثقة
  ├─ □ Post-mortem للحوادث الكبرى
  ├─ □ إجراءات تصحيحية
  └─ □ متابعة التنفيذ
```

**الأدلة المطلوبة:**
- [ ] خطة الاستجابة للحوادث
- [ ] سجلات التمارين
- [ ] تقارير الحوادث السابقة
- [ ] تقارير Post-mortem

### 4.4 استمرارية الأعمال (Business Continuity)

```
□ خطة استعادة الكوارث
  ├─ □ DRP موثقة ومحدثة
  ├─ □ RTO/RPO محددة لكل نظام
  ├─ □ مواقع بديلة جاهزة
  └─ □ إجراءات Failover موثقة

□ النسخ الاحتياطي
  ├─ □ استراتيجية 3-2-1
  ├─ □ نسخ مشفرة
  ├─ □ اختبار الاستعادة دوري
  └─ □ Off-site storage

□ اختبار الاستعادة
  ├─ □ اختبار النسخ الاحتياطية (شهري)
  ├─ □ اختبار DR جزئي (ربع سنوي)
  ├─ □ اختبار DR كامل (سنوي)
  └─ □ توثيق النتائج
```

**الأدلة المطلوبة:**
- [ ] خطة DR
- [ ] سجلات النسخ الاحتياطي
- [ ] تقارير اختبار الاستعادة
- [ ] إحصائيات RTO/RPO الفعلية
